# Actuator Mini MES 요구사항 명세서

| 항목 | 내용 |
|---|---|
| 문서 버전 | `v0.1` |
| 대상 버전 | Mini MES 1차 버전 |
| 작성일 | 2026-07-23 |
| 구현 기술 | Python, SQLite, Streamlit |
| 생산 대상 | 자동차 HVAC 에어믹스 도어 액추에이터 |
| 문서 상태 | 초안 |

## 1. 문서 목적

이 문서는 Actuator Mini MES 1차 버전에서 구현할 기능, 업무 규칙, 데이터 및 완료 조건을 정의합니다. 이후 ERD, 테이블, Python 로직 및 Streamlit 화면은 이 문서를 기준으로 설계합니다.

## 2. 시스템 범위

시스템은 가상 생산라인 1개에서 제품 1종을 생산하는 상황을 대상으로 합니다. 사용자는 화면을 통해 기준정보와 생산 데이터를 직접 등록하고 생산 현황과 추적 결과를 조회합니다.

### 2.1 포함 범위

1. 품목 관리
2. BOM 관리
3. 공정 및 Routing 관리
4. 자재 LOT 입고
5. 작업지시 생성
6. 완제품 Serial Number 발급
7. 공정별 작업실적 등록
8. 자재 LOT 투입 및 사용 등록
9. EOL 검사 결과 등록
10. 생산 완료 및 상태 관리
11. 생산 진행 현황 조회
12. Serial Number 및 자재 LOT 기준 양방향 추적

### 2.2 제외 범위

- 인증, 로그인 및 사용자 권한
- 작업자·설비 상세 기준정보
- 구매발주, 창고 이동, 출하 및 복잡한 재고관리
- 재작업, 재검사, 폐기 및 보류 처리
- 실제 바코드·QR 스캐너 연동
- PLC·센서·설비 실시간 통신
- OEE, SPC 및 고급 통계 분석
- ERP 및 외부 시스템 연동
- AI 불량 예측 및 예지보전
- Docker, FastAPI 및 서버 배포

## 3. 사용자 역할

1차 버전은 로그인과 권한을 구현하지 않습니다. 다만 화면과 업무를 이해하기 위해 다음 역할을 가정합니다.

| 역할 | 주요 업무 |
|---|---|
| 관리자 | 품목, BOM, 공정, Routing 기준정보 관리 |
| 자재 담당자 | 자재 LOT 입고 등록 |
| 생산 관리자 | 작업지시 생성, Serial Number 발급, 진행 현황 확인 |
| 작업자 | 공정실적과 투입 자재 LOT 등록 |
| 품질 담당자 | EOL 검사값과 최종 판정 확인 |

## 4. 공통 업무 규칙

- 모든 품목코드, 작업지시번호, 자재 LOT 번호 및 완제품 Serial Number는 중복될 수 없습니다.
- 삭제로 생산이력이 끊어지지 않도록, 이력이 연결된 기준정보와 작업지시는 원칙적으로 물리 삭제하지 않습니다.
- 날짜와 시간은 데이터베이스에 일관된 형식으로 저장합니다.
- 수량은 0보다 커야 하며, 사용 수량은 해당 LOT의 가용 수량을 초과할 수 없습니다.
- 완제품은 지정된 Routing 순서대로 공정을 진행해야 합니다.
- 선행 공정이 완료되지 않으면 다음 공정을 완료할 수 없습니다.
- EOL 검사 PASS 제품만 생산 완료 상태가 될 수 있습니다.
- 1차 버전에서 FAIL 제품은 불합격 상태로 기록하며, 이후 재작업 흐름은 처리하지 않습니다.

## 5. 기능 요구사항

### FR-01 품목 관리

완제품과 생산에 사용되는 자재의 기준정보를 등록하고 조회합니다.

필수 입력 항목:

- 품목코드
- 품목명
- 품목구분: `PRODUCT`, `MATERIAL`
- 사용 여부

처리 규칙:

- 품목코드는 고유해야 합니다.
- BOM이나 생산이력에서 사용 중인 품목은 삭제하지 않습니다.

완료 조건:

- 완제품과 자재를 구분하여 등록하고 목록에서 조회할 수 있습니다.

### FR-02 BOM 관리

완제품 1개 생산에 필요한 자재와 기준 수량을 정의합니다.

필수 입력 항목:

- 완제품 품목코드
- 자재 품목코드
- 소요 수량
- 단위

처리 규칙:

- BOM의 상위 품목은 `PRODUCT`, 구성 품목은 `MATERIAL`이어야 합니다.
- 동일 완제품에 같은 자재를 중복 등록할 수 없습니다.
- 소요 수량은 0보다 커야 합니다.

완료 조건:

- 제품별 BOM을 등록하고 구성 자재와 필요 수량을 조회할 수 있습니다.

### FR-03 공정 및 Routing 관리

표준 공정과 제품이 거쳐야 할 공정 순서를 정의합니다.

필수 입력 항목:

- 공정코드
- 공정명
- 제품코드
- 공정 순번
- 사용 여부

처리 규칙:

- 같은 제품에서 공정 순번은 중복될 수 없습니다.
- 1차 대상 제품의 Routing은 `OP10`부터 `OP100`까지로 구성합니다.
- 기준정보는 초기 데이터로 제공하되 화면에서 조회할 수 있어야 합니다.

완료 조건:

- 대상 제품의 전체 공정 순서를 조회할 수 있습니다.

### FR-04 자재 LOT 입고

생산에 사용할 자재의 입고 LOT와 수량을 등록합니다.

필수 입력 항목:

- 자재 LOT 번호
- 자재 품목코드
- 입고일자
- 입고수량

선택 입력 항목:

- 공급업체명 또는 공급업체 코드
- 비고

처리 규칙:

- 자재 LOT 번호는 고유해야 합니다.
- `MATERIAL` 품목만 입고할 수 있습니다.
- 입고수량은 0보다 커야 합니다.
- 가용 수량은 입고수량에서 누적 사용수량을 뺀 값으로 계산합니다.

완료 조건:

- LOT를 등록하고 품목별 입고수량·사용수량·가용수량을 조회할 수 있습니다.

### FR-05 작업지시 생성

생산할 제품, 수량 및 일정을 지정한 작업지시를 생성합니다.

필수 입력 항목:

- 작업지시번호
- 제품코드
- 계획수량
- 계획일자

처리 규칙:

- 작업지시번호는 고유해야 합니다.
- `PRODUCT` 품목만 생산 대상으로 선택할 수 있습니다.
- 계획수량은 0보다 커야 합니다.
- 생성 시 상태는 `PLANNED`입니다.
- 하나 이상의 Serial Number가 발급되면 `IN_PROGRESS`로 변경합니다.
- 계획수량만큼 PASS 완료되면 `COMPLETED`로 변경합니다.

완료 조건:

- 작업지시를 생성하고 상태와 계획·진행·완료 수량을 조회할 수 있습니다.

### FR-06 완제품 Serial Number 발급

작업지시에 종속된 개별 완제품 식별번호를 자동으로 발급합니다.

권장 형식:

`HVA-YYYYMMDD-000001`

처리 규칙:

- Serial Number는 시스템 전체에서 고유해야 합니다.
- 작업지시의 계획수량을 초과하여 발급할 수 없습니다.
- 발급된 제품의 최초 상태는 `CREATED`입니다.
- Serial Number는 생산 시작 전에 발급하여 전 공정의 이력을 연결하는 식별자로 사용합니다.

완료 조건:

- 지정 수량의 Serial Number를 중복 없이 일괄 발급하고 작업지시별로 조회할 수 있습니다.

### FR-07 공정별 작업실적 등록

Serial Number별 공정 시작·완료 시각과 처리 결과를 기록합니다.

필수 입력 항목:

- Serial Number
- 공정코드
- 시작일시
- 완료일시
- 공정 결과: `PASS`, `FAIL`

처리 규칙:

- Routing에 존재하는 공정만 등록할 수 있습니다.
- 완료일시는 시작일시보다 빠를 수 없습니다.
- 첫 공정이 시작되면 제품 상태를 `IN_PROGRESS`로 변경합니다.
- 선행 공정이 완료되지 않으면 후속 공정을 완료할 수 없습니다.
- 같은 Serial Number와 공정의 완료 실적은 중복 등록할 수 없습니다.

완료 조건:

- 제품별 완료 공정, 현재 공정, 다음 공정 및 공정별 소요시간을 확인할 수 있습니다.

### FR-08 자재 LOT 투입 및 사용 등록

특정 완제품에 실제로 사용한 자재 LOT와 수량을 연결합니다.

필수 입력 항목:

- Serial Number
- 공정코드
- 자재 LOT 번호
- 사용수량
- 투입일시

처리 규칙:

- BOM에 포함된 자재만 투입할 수 있습니다.
- 선택한 LOT의 자재 품목이 BOM의 자재와 일치해야 합니다.
- 사용수량은 LOT 가용수량을 초과할 수 없습니다.
- 동일한 투입 기록이 중복 저장되지 않아야 합니다.

완료 조건:

- 완제품별 투입 자재 LOT를 조회하고 LOT별 누적 사용량과 가용수량을 계산할 수 있습니다.

### FR-09 EOL 검사 결과 등록

완제품의 최종 기능검사 측정값과 판정을 기록합니다.

1차 검사 항목:

| 검사 항목 | 저장 값 | 판정 방식 |
|---|---|---|
| 동작 방향 | 정방향·역방향 결과 | 두 방향 모두 정상 |
| 동작시간 | 초 단위 측정값 | 기준 범위 비교 |
| 최종 위치 | 각도 또는 센서 전압 | 기준 범위 비교 |
| 최대 소비전류 | A 또는 mA 단위 측정값 | 상한 기준 비교 |

처리 규칙:

- `OP90` 이전의 필수 공정이 완료되어야 검사할 수 있습니다.
- 항목별 측정값과 판정을 저장합니다.
- 모든 필수 항목이 PASS이면 최종 판정은 `PASS`입니다.
- 하나 이상의 필수 항목이 FAIL이면 최종 판정은 `FAIL`입니다.
- 구체적인 수치 기준은 데이터베이스 설계 전에 별도로 확정합니다.

완료 조건:

- Serial Number별 측정값, 항목 판정 및 최종 판정을 조회할 수 있습니다.

### FR-10 생산 완료 및 상태 관리

작업지시와 완제품의 현재 상태를 관리합니다.

완제품 상태:

| 상태 | 의미 |
|---|---|
| `CREATED` | Serial Number 발급 완료 |
| `IN_PROGRESS` | 하나 이상의 생산공정 진행 중 |
| `PASS` | EOL 합격 및 필수 공정 완료 |
| `FAIL` | EOL 최종 불합격 |

작업지시 상태:

| 상태 | 의미 |
|---|---|
| `PLANNED` | 작업지시 생성 |
| `IN_PROGRESS` | Serial 발급 또는 생산 진행 중 |
| `COMPLETED` | 계획수량 생산 완료 |
| `CANCELLED` | 작업지시 취소 |

처리 규칙:

- EOL 최종 판정이 PASS이고 필수 공정이 완료된 제품만 `PASS`로 변경합니다.
- EOL 최종 판정이 FAIL이면 제품을 `FAIL`로 변경합니다.
- 상태 변경은 관련 업무 처리 결과에 따라 시스템이 수행합니다.

완료 조건:

- 제품과 작업지시의 현재 상태가 실제 등록 이력과 일치합니다.

### FR-11 생산 진행 현황 조회

작업지시별 생산 진행 상황을 집계하여 보여줍니다.

조회 항목:

- 작업지시번호와 제품
- 계획수량
- Serial 발급수량
- 생산 진행수량
- PASS 수량
- FAIL 수량
- 달성률
- 공정별 WIP 수량

처리 규칙:

- 달성률은 `PASS 수량 / 계획수량 × 100`으로 계산합니다.
- 수량은 저장된 Serial Number와 상태를 기준으로 집계합니다.

완료 조건:

- 작업지시를 선택하면 최신 생산수량과 진행률을 확인할 수 있습니다.

### FR-12 생산이력 및 양방향 추적

완제품과 자재 사이의 생산 계보를 조회합니다.

완제품 기준 조회:

- Serial Number와 현재 상태
- 작업지시 정보
- 공정별 시작·완료 일시와 결과
- 투입 자재 품목, LOT 및 수량
- EOL 검사 측정값과 최종 판정

자재 LOT 기준 조회:

- LOT 기본정보와 입고·사용·가용수량
- 해당 LOT가 투입된 완제품 Serial Number 목록
- 각 완제품의 작업지시, 상태 및 EOL 판정

완료 조건:

- Serial Number 하나로 해당 제품의 전체 생산이력을 조회할 수 있습니다.
- 자재 LOT 하나로 영향을 받는 모든 완제품을 누락 없이 조회할 수 있습니다.

## 6. 화면 요구사항

| 화면 | 주요 기능 |
|---|---|
| 메인 대시보드 | 작업지시별 계획, 진행, PASS, FAIL 및 달성률 조회 |
| 품목 관리 | 완제품·자재 등록 및 조회 |
| BOM 관리 | 제품별 구성 자재와 필요 수량 등록 및 조회 |
| 공정·Routing | 표준 공정과 제품별 순서 조회 |
| 자재 LOT 입고 | LOT와 입고수량 등록, 가용수량 조회 |
| 작업지시 | 작업지시 생성, 상태 및 실적 조회 |
| Serial 발급 | 작업지시별 Serial Number 일괄 발급 및 조회 |
| 생산실적 | 공정 시작·완료 및 자재 LOT 투입 등록 |
| EOL 검사 | 검사값 입력, 항목별·최종 판정 표시 |
| 생산이력 추적 | Serial Number 또는 자재 LOT 기준 추적 |

## 7. 비기능 요구사항

### NFR-01 사용성

- 모든 화면과 메시지는 초보 사용자도 이해할 수 있는 한국어로 표시합니다.
- 필수 입력값, 잘못된 값 및 처리 결과를 명확히 안내합니다.
- 사용자가 식별번호를 직접 입력할 때 오타를 줄일 수 있도록 선택 상자나 검색 기능을 우선 사용합니다.

### NFR-02 데이터 무결성

- SQLite 외래키 제약을 활성화합니다.
- 고유 식별자는 `UNIQUE` 제약으로 중복을 방지합니다.
- 필수값은 `NOT NULL`, 상태값은 `CHECK` 또는 업무 로직으로 제한합니다.
- 여러 데이터가 함께 변경되는 업무는 트랜잭션으로 처리합니다.

### NFR-03 유지보수성

- Streamlit 화면, DB 연결, SQL 조회 및 업무 규칙을 가능한 한 분리합니다.
- SQL문은 파라미터 바인딩을 사용합니다.
- DB 경로는 실행 위치가 아닌 프로젝트 파일 위치를 기준으로 계산합니다.
- 상태값과 식별번호 생성 규칙은 코드 여러 곳에 중복 작성하지 않습니다.

### NFR-04 실행 환경

- 개인 노트북의 Windows 11 및 WSL 환경에서 실행할 수 있어야 합니다.
- 별도 DB 서버 없이 SQLite 파일로 실행할 수 있어야 합니다.
- 초기 DB는 `schema.sql`과 `seed.sql`로 재생성할 수 있어야 합니다.

### NFR-05 성능

- 학습용 샘플 데이터 규모에서 등록과 일반 조회는 사용자가 지연을 크게 느끼지 않는 수준이어야 합니다.
- 추적 조회에 사용되는 Serial Number, LOT 번호 및 외래키에는 필요한 인덱스를 적용합니다.

## 8. 검증 시나리오

### 시나리오 A: 정상 생산

1. 완제품과 BOM 자재를 등록합니다.
2. 제품 BOM과 Routing을 준비합니다.
3. BOM 자재의 LOT를 입고합니다.
4. 계획수량 3개의 작업지시를 생성합니다.
5. 완제품 Serial Number 3개를 발급합니다.
6. 각 제품의 공정실적과 자재 투입을 등록합니다.
7. EOL 검사값을 입력하고 모두 PASS로 판정합니다.
8. 제품 3개와 작업지시가 완료 상태인지 확인합니다.

### 시나리오 B: 검사 불합격

1. 정상 생산과 동일하게 공정실적을 등록합니다.
2. EOL 검사 항목 하나에 기준을 벗어난 값을 입력합니다.
3. 해당 항목과 최종 판정이 FAIL인지 확인합니다.
4. 제품 상태가 FAIL이고 작업지시의 불합격 수량이 증가했는지 확인합니다.

### 시나리오 C: 완제품 이력 추적

1. Serial Number를 검색합니다.
2. 작업지시, 전 공정 실적, 투입 자재 LOT 및 EOL 결과가 함께 조회되는지 확인합니다.

### 시나리오 D: 자재 LOT 영향 추적

1. 여러 완제품에 동일한 모터 LOT를 투입합니다.
2. 해당 모터 LOT를 검색합니다.
3. LOT가 사용된 모든 완제품과 각 제품의 상태·검사 결과가 조회되는지 확인합니다.

### 시나리오 E: 업무 규칙 차단

- 계획수량 초과 Serial 발급
- 재고보다 많은 자재 사용
- BOM에 없는 자재 투입
- 선행 공정 미완료 상태에서 후속 공정 완료
- 필수 공정 미완료 상태에서 EOL 검사 또는 생산 완료

위 동작을 시도했을 때 저장되지 않고 원인을 설명하는 메시지가 표시되어야 합니다.
